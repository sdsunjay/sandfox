# Sandfox

Hardened Firefox browser running in a Docker container on macOS with X11 display forwarding and PulseAudio for audio.

## Prerequisites

- **Colima** â€” lightweight Docker runtime (`brew install colima docker docker-compose docker-buildx`)
- **XQuartz** â€” X11 server for macOS (`brew install --cask xquartz`). Requires logout/login after first install.
- **PulseAudio** â€” audio server (`brew install pulseaudio`)

## Starting all services

The easiest way to start everything is with `run.sh`, which handles all prerequisites automatically:

```bash
bash run.sh start
```

`run.sh` performs these steps in order, skipping any service that is already running:

1. Starts Colima (Docker runtime)
2. Starts XQuartz (X11 display server)
3. Generates an X11 auth cookie (MIT-MAGIC-COOKIE) for secure display forwarding
4. Starts PulseAudio with cookie-based TCP auth on port 4713
5. Configures network egress filtering in the Colima VM (iptables)
6. Runs `docker compose up --build -d`
7. Starts session logging to `~/.config/sandfox/logs/`

**Important:** Always use `bash run.sh start` â€” do not run `docker compose up` directly. The run script generates auth files and configures network filtering that the container depends on.

## Stopping services

```bash
bash run.sh stop
```

## Checking status

```bash
bash run.sh status
```

## Architecture

### Container setup

- **Base image**: Ubuntu 22.04
- **Firefox**: Installed from Mozilla PPA (not the snap stub)
- **User**: Non-root `firefox` user (UID/GID 999), member of `audio` and `video` groups
- **Filesystem**: Read-only rootfs with tmpfs mounts for writable paths
- **Entrypoint**: `entrypoint.sh` copies the hardened profile from `/opt/firefox-profile` into the writable tmpfs at `~/.mozilla`, configures PulseAudio client to use the mounted auth cookie, then launches Firefox

### Display (X11)

- `DISPLAY=host.docker.internal:0` forwards to XQuartz on the host
- `LIBGL_ALWAYS_SOFTWARE=1` forces software rendering (no GPU passthrough)
- `MOZ_ENABLE_WAYLAND=0` ensures X11 mode
- X11 authentication uses MIT-MAGIC-COOKIE (Xauth) â€” the cookie is generated by `run.sh` and bind-mounted into the container. `xhost +localhost` is no longer used.

### Audio (PulseAudio)

- `PULSE_SERVER=tcp:host.docker.internal:4713` connects to the host PulseAudio over TCP
- PulseAudio uses cookie-based authentication (`auth-anonymous=0`). The cookie is generated by `run.sh` and bind-mounted into the container.
- The entrypoint configures `~/.config/pulse/client.conf` to point at the mounted cookie file.

### Security hardening

- `read_only: true` â€” immutable root filesystem
- `cap_drop: ALL`, then selectively adds `SYS_CHROOT` only (Firefox uses unprivileged user namespaces)
- `no-new-privileges: true` â€” prevents privilege escalation
- Custom seccomp profile (`seccomp-firefox.json`) â€” default-deny allowlist; blocks `io_uring`, `syslog`, `name_to_handle_at`, and other exploit/escape vectors
- Network egress filtering â€” iptables rules in Colima VM restrict outbound to HTTP/HTTPS/DNS only; blocks all RFC 1918 ranges and host gateway (except X11 port 6000 and PulseAudio port 4713)
- `pids_limit: 512` â€” prevents fork bombs
- `ulimits: core=0` â€” disables core dumps (prevents memory content leaks)
- `noexec,nosuid,nodev` on tmpfs mounts â€” prevents execution of dropped binaries
- DNS-over-HTTPS via Cloudflare's malware-blocking resolver (`security.cloudflare-dns.com`)
- Safe Browsing disabled for OPSEC â€” prevents leaking investigated URLs to Google
- Resource limits: 6 CPUs, 6 GB memory

### Auth files

Generated by `run.sh` and stored at `~/.config/sandfox/`:

| File            | Purpose                                          |
|-----------------|--------------------------------------------------|
| `xauth`         | MIT-MAGIC-COOKIE for X11 display authentication |
| `pulse-cookie`  | PulseAudio cookie for TCP audio authentication  |
| `logs/`         | Session logs from container stdout/stderr        |

### tmpfs mounts

| Mount                      | Size  | Flags                    | Purpose                        |
|----------------------------|-------|--------------------------|--------------------------------|
| `/tmp`                     | 256m  | noexec,nosuid,nodev      | General temp files             |
| `/run`                     | 64m   | noexec,nosuid,nodev      | Runtime files                  |
| `/home/firefox/.cache`     | 128m  | noexec,nosuid,nodev      | Browser cache                  |
| `/home/firefox/.mozilla`   | 256m  | nosuid,nodev             | Firefox profile (copied on start) |
| `/home/firefox/.config`    | 8m    | noexec,nosuid,nodev      | PulseAudio client config       |
| `/dev/shm`                 | 256m  | noexec,nosuid,nodev      | Shared memory (Firefox IPC)    |

### Pre-installed extensions and policies

- **uBlock Origin** installed via `distribution/extensions/` at build time
- **Enterprise policies** (`policies.json`) configure Firefox settings at the distribution level
- **Hardened profile** (`user.js`) sets privacy/security-focused `about:config` prefs

## Troubleshooting

### Quick diagnostic commands

Run these to check the state of all services at a glance:

```bash
bash run.sh status
```

Or manually:

```bash
# Colima / Docker status
colima status
docker info 2>&1 | head -5

# XQuartz running?
pgrep -l Xquartz

# X11 auth cookie exists?
ls -la ~/.config/sandfox/xauth

# PulseAudio listening on TCP 4713?
lsof -iTCP:4713 -sTCP:LISTEN

# PulseAudio cookie exists?
ls -la ~/.config/sandfox/pulse-cookie

# Network egress rules active?
colima ssh -- sudo iptables -L SANDFOX-FILTER -n 2>/dev/null

# Container status and logs
docker compose ps
docker compose logs sandfox
```

### Common problems

- **"Cannot connect to the Docker daemon"** or **Docker socket not found**: Colima is not running. Run `colima start` and wait for it to finish before retrying.
- **No display / Firefox window doesn't appear**:
  1. Check that XQuartz is running: `pgrep -l Xquartz`. If not, run `open -a XQuartz` and wait a couple of seconds.
  2. Check that the X11 auth cookie exists: `ls -la ~/.config/sandfox/xauth`. If missing or empty, re-run `bash run.sh start`.
  3. If cookie auth fails, the script falls back to `xhost +localhost` automatically.
- **No audio**:
  1. Verify PulseAudio is listening: `lsof -iTCP:4713 -sTCP:LISTEN`.
  2. Verify the cookie exists: `ls -la ~/.config/sandfox/pulse-cookie`.
  3. Re-run `bash run.sh start` to regenerate the cookie and restart PulseAudio.
- **Container exits immediately**: Check logs with `docker compose logs sandfox`. Common causes:
  - X11 connection refused â€” re-run `bash run.sh start`.
  - Seccomp violation â€” if Firefox hits a blocked syscall, check logs and consider adding the syscall to `seccomp-firefox.json`.
  - Out of tmpfs space â€” the container uses tmpfs for all writable paths; heavy browsing can fill them.
- **Firefox sandbox warnings**: If Firefox logs sandbox-related errors, this may be because `SYS_ADMIN` was removed. The container itself is the sandbox boundary â€” Firefox's internal sandbox failing is acceptable. If you need Firefox's content sandbox, add `SYS_ADMIN` back to `cap_add` in `docker-compose.yml`.
- **libGL errors**: Expected and harmless â€” software rendering is used (`LIBGL_ALWAYS_SOFTWARE=1`), there is no GPU passthrough.
- **DBus warnings**: Harmless â€” `dbus-x11` is not installed and no dbus daemon runs in the container.

### When helping a user who is stuck

If the user says "it's not working" without more detail, run `bash run.sh status` to identify which service is down. The simplest fix is to run `bash run.sh start`, which is idempotent and will start/configure any missing services.
